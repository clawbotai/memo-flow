## 核心组件
- 应用外壳 AppShell：统一承载侧边栏与主内容区，并协调设置面板的弹窗开关。
- 侧边栏 Sidebar：提供桌面端固定侧栏与移动端抽屉式菜单，支持导航切换与设置入口。
- Whisper 设置面板 WhisperSettings：负责加载/保存配置、监听下载进度、展示状态与高级设置。
- 转录历史卡片 TranscriptionCard：展示转录记录的概要信息，包括状态徽章、进度条、字数统计和时间戳。
- 转录详情视图 TranscriptionDetail：提供实时进度跟踪、音频播放、逐字稿展示和重新转录功能，具备自动滚动、响应式布局和改进的实时进度显示。
- 播客页面 PodcastPage：提供播客转录功能，包含新的 `connectToTranscribeProgress` 函数、更好的状态管理和自动重连功能。
- 转录历史管理：基于临时文件系统的转录记录管理，支持增删改查和状态同步。
- 类型系统与工具：WhisperConfig/WhisperStatus 定义数据结构；whisper-config 提供配置持久化与环境变量覆盖；whisper 提供转写封装；xiaoyuzhou 提供播客信息提取；transcription-task-manager 提供任务状态管理。

## 架构总览
MemoFlow 采用"UI 组件 + 服务端 API 路由 + 本地配置/文件"的分层架构：
- UI 层：AppShell/Sidebar/WhisperSettings/TranscriptionCard/TranscriptionDetail/PodcastPage 负责界面与交互
- 业务层：API 路由封装配置读取/保存、状态查询、模型下载与进度推送、播客处理、转录历史管理
- 数据层：本地文件系统存储配置与模型，SSE 实时推送下载进度，临时目录存储转录进度文件
- 类型层：统一的数据契约 WhisperConfig/WhisperStatus/TranscriptionRecord/TranscribeProgress

## 详细组件分析

### 播客页面（PodcastPage）- 重大改进
- 职责边界
  - 提供播客转录功能，支持小宇宙播客链接输入与处理
  - 实时展示转录进度和状态，包含阶段指示器、进度条和实时文稿
  - 管理转录历史记录的加载与展示
  - 处理音频播放、转录结果展示和文件保存
- 状态管理
  - 本地状态：podcastUrl、podcastTranscript、podcastAudioInfo、taskId、transcribeStage、transcribeStatus、liveSegments、transcribeProgress、episodeTitle、savedPath、finalSegments、transcriptionHistory、historyLoading
  - 引用状态：transcribeEsRef、transcribeReconnectTimeoutRef、activeTaskIdRef、transcribeFinishedRef、scrollRef
  - 生命周期：建立 SSE 连接、自动滚动到底部、错误重连机制、任务清理
- 用户交互
  - 输入播客链接并提交转录请求
  - 实时查看转录进度和阶段状态
  - 自动滚动到最新内容
  - 点击重新转录按钮触发后台处理
  - 复制转录结果到剪贴板
- 数据流
  - 读取：GET /api/transcription-history -> 历史记录列表
  - 写入：POST /api/process-podcast -> 启动转录任务
  - 进度：SSE /api/transcribe-progress -> 实时进度推送
  - 清理：组件卸载时清理 EventSource 和定时器

**更新** 新增了 `connectToTranscribeProgress` 函数，提供更好的状态管理和自动重连功能

#### connectToTranscribeProgress 函数
- 职责：建立与转录进度 API 的 SSE 连接，处理实时数据更新和错误重连
- 状态管理：使用 `activeTaskIdRef` 确保正确的任务标识，使用 `transcribeFinishedRef` 防止过期任务更新
- 自动重连：使用 `transcribeReconnectTimeoutRef` 和 `clearReconnectTimer` 实现智能重连机制
- 数据处理：解析 JSON 数据，更新多个状态字段，包括阶段、状态、片段、进度、音频URL等
- 生命周期：清理旧连接，处理连接错误，自动关闭完成的任务

#### 改进的自动重连机制
- 使用 `setTimeout` 实现延迟重连，避免频繁重试
- 检查任务完成状态和任务ID一致性，防止过期数据更新
- 清理定时器和 EventSource，避免内存泄漏
- 支持断线自动恢复，提升用户体验

## 性能考虑
- 并发加载：Whisper 设置面板在对话框打开时并发请求状态与配置，减少等待时间
- SSE 进度：使用 Server-Sent Events 实时推送下载进度，避免轮询带来的额外开销
- 临时文件清理：转写完成后清理输出文件，避免磁盘占用累积
- 环境变量优先：通过环境变量覆盖配置，避免重复 IO 与解析
- 响应式布局：桌面端固定侧栏，移动端抽屉式菜单，降低 DOM 重排成本
- 进度文件合并：实时合并进度文件数据，避免频繁数据库读写
- 自动滚动优化：使用 requestAnimationFrame 优化滚动性能
- 错误重连机制：SSE 断线自动重连，提升用户体验
- **新增**：connectToTranscribeProgress 函数提供更精确的任务状态管理，避免过期任务更新
- **新增**：智能重连机制使用 setTimeout 和 clearReconnectTimer，避免频繁重试
- **新增**：引用状态管理（activeTaskIdRef、transcribeFinishedRef）确保任务隔离
